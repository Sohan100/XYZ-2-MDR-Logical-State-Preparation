Introduction

Quantum computing’s utility lies in the usage of superposition and entanglement to process information,
which is an advantage that makes it distinct from classical computing. Classical computers can manipulate
bits, b ∈ {0, 1} with irreversible Boolean gates—e.g. the AND gate yields AND(1, 0) = 0, from which the
inputs cannot be reconstructed.
Quantum computers, by contrast, rely on qubits whose orthonormal basis states, |0⟩ and |1⟩, reside in
coherent superposition and become entangled across multiple physical quantum units [1]. Operations on
these qubits are conducted through unitary quantum gates, which are reversible. These unitary operations
allow quantum computers to differ qualitatively to classical computers in the domains of state space, gate
design, algorithmic speed-ups, and especially error correction. These fundamental distinctions enlarge the
accessible state space exponentially, enabling algorithmic speed-ups such as Shor’s factoring and Grover’s
search, and impose unique demands on preserving fragile quantum coherence.
Classical errors are discrete bit flips that measurement easily reveals. Quantum noise is continuous: stray
fields rotate states, and entanglement with the environment erases phase. Direct measurement would destroy
the superposed information, so errors must be detected indirectly. So, a critical step towards building a large
scale fault-tolerant quantum computer is quantum error correction.
Fault-tolerant quantum processing units (QPUs) suppress noise by encoding each logical qubit into a
block of physical qubits using a quantum error correcting code (QECC). Stabilizer codes realize this by
repeatedly measuring a set of mutually commuting operators—the syndrome—whose outcomes map the underlying continuous noise to discrete Pauli errors that can be classically decoded and corrected [2].Topological
variants, notably Kitaev’s surface code, are especially appealing: their stabilizers act on geometrically local
qubits in a 2-dimensional grid, and, for physical error rates below threshold, the logical error probability
falls exponentially with code distance [3]. Recent experiments have successfully run surface-code cycles
and other stabilizer codes on superconducting, trapped-ion, and photonic platforms [4, 5]. Iterating syndrome extraction and decoding thus drives logical error rates to levels compatible with largescale quantum
algorithms.

Matching codes

be viewed as a union of ideas coming from the surface code and color code [7]. They are defined on any
trivalent lattice, where the qubits sit on vertices, and the links connecting the qubits are defined by X, Y ,
or Z Pauli operators. These links are coloured so that no two adjacent links share the same label. For each

Isolated single-qubit errors trigger two neighboring plaquette defects aligned along one of three principal
lattice directions, where the direction is determined by the Pauli type (X, Y , or Z). Thus, the syndrome
indicates a specific Pauli error class, generalizing the two-direction structure of XZZX to three directions
for the XY Z 2 code [6, 9]. Link stabilizers produce single-parity violations on a qubit pair, which must be
handled in conjunction with the plaquette defects. Although each plaquette is weight 6, the hexagonal tiling
yields fewer high-weight checks than square-lattice deformations (e.g. XZZX has more weight-4 plaquettes
for a comparable number of data qubits), partly offsetting the cost of higher weight with a lower check density [6]. The many weight-2 link checks are also attractive for frequent and reliable measurement. However,
investigating how this code behaves on hardware whilst discovering which noise biases work best with it will
give us a fair assessment of the code’s performance and capabilities.

MDR using Tableau Simulations

With these noise channels in mind, we conduct the MDR state preparation protocol with 3 different types
of noise models. The noise models we use are the (1) unbiased depolarizing noise, (2) Z type noise and (3)
pure Z noise for both single qubit and 2-qubit channels. These noise models are simulated in O(n2 ) time,
using a tableau simulator via the stim python package. As every gate and noise process used in the MDR
circuit is Pauli-diagonal, stim can track an n-qubit stabiliser tableau in O(n2 ) time per two-qubit operation,
as opposed to an exponential runtime with a state vector simulator.
Initially, we simulate the MDR protocol on the distance 3 case of the XY Z 2 code. We construct our
simulations to calculate the fidelities of our stabilizers and logical operators after multiples rounds of MDR.
The fidelity is calculated as the expectation value of a specific operator. At the end of a designated round of
MDR, we destructively measure the concerned operator and calculate the expectation value using Equation 9.
Multiple rounds of MDR are simulated to investigate any performance changes of the operator fidelities.
